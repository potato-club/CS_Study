# 📌 페이징(Paging)과 세그먼테이션(Segmentation)

## 기법을 사용하는 이유
다중 프로그래밍 시스템에서 여러 프로세스를 수용하기 위해  
**주기억장치를 동적으로 분할**하는 메모리 관리 작업이 필요하다.

## 메모리 관리 기법

### 1. 연속 메모리 관리
프로그램 전체가 하나의 큰 공간에 **연속적으로 할당**되어야 한다.

- **고정 분할 기법**: 메모리를 고정된 크기의 파티션으로 나눔  
  → **내부 단편화 발생**
- **동적 분할 기법**: 필요한 만큼 메모리를 동적으로 할당  
  → **외부 단편화 발생**

### 2. 불연속 메모리 관리
프로그램이 여러 조각으로 나뉘어 **서로 다른 주소 공간에 할당** 가능

- **페이지(Page)**: 고정 크기의 프로세스 조각
- **프레임(Frame)**: 페이지와 같은 크기의 메모리 조각
- **세그먼트(Segment)**: 논리적으로 나뉜 가변 크기 블록

## 단편화 개념
- **내부 단편화**: 할당받은 공간 중 사용하지 못하는 남는 공간
- **외부 단편화**: 사용 가능한 메모리 공간이 흩어져 있어 할당 불가

## 고정 크기: 페이징 (Paging)

### 단순 페이징
- 프로세스는 **고정 크기의 페이지**로 나뉘고, 메모리는 프레임으로 나뉨
- 각 페이지는 프레임에 매핑됨
- **외부 단편화 없음**
- **일부 내부 단편화 존재**

### 가상 메모리 페이징
- 필요한 페이지만 메모리에 적재 (나머지는 나중에 자동 로드)
- **외부 단편화 없음**
- **오버헤드 증가** (페이지 교체, 페이지 폴트 등으로 인한)


## 가변 크기: 세그먼테이션 (Segmentation)

### 단순 세그먼테이션
- 프로세스는 논리적으로 나뉜 **여러 개의 세그먼트**로 구성됨
- 세그먼트는 가변 크기
- **내부 단편화 없음**
- **외부 단편화 존재**

### 가상 메모리 세그먼테이션
- 필요한 세그먼트만 메모리에 적재
- **내부 단편화 없음**
- **오버헤드 존재** (세그먼트 테이블, 주소 변환 등)

---

## 요약
- **페이징**: 고정 크기 단위, 외부 단편화 없음, 일부 내부 단편화 있음
- **세그먼테이션**: 가변 크기 단위, 내부 단편화 없음, 외부 단편화 있음
- **가상 메모리** 기법은 실제 메모리에 전부 적재하지 않아도 됨 (지연 로딩)
